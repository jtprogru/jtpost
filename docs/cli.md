# Документация CLI

## Обзор

**jtpost** — CLI-утилита для управления жизненным циклом постов в Telegram-канале.

## Глобальные флаги

Эти флаги доступны для всех команд:

| Флаг | Короткий | Описание | По умолчанию |
|------|----------|----------|--------------|
| `--config` | `-c` | Путь к конфигурационному файлу | `.jtpost.yaml` |
| `--posts-dir` | `-D` | Директория с постами (переопределяет конфиг) | из конфига |
| `--verbose` | `-v` | Подробный вывод | `false` |
| `--help` | `-h` | Показать справку | - |
| `--version` | - | Показать версию | - |

## Команды

### init

Инициализация проекта — создаёт конфигурационный файл и директории.

```bash
jtpost init
```

**Что делает:**
- Создаёт `.jtpost.yaml` с настройками по умолчанию
- Создаёт директорию для постов (`content/posts`)
- Создаёт директорию для шаблонов (`templates`)

**Пример вывода:**
```
✅ Проект jtpost инициализирован!

📁 Конфигурация: /path/to/.jtpost.yaml
📁 Директория постов: content/posts
📁 Директория шаблонов: templates

📝 Следующий шаг: создайте первый пост командой
   jtpost new "Заголовок поста"
```

### new

Создание нового поста с указанным заголовком.

```bash
jtpost new "Заголовок поста" [флаги]
```

**Флаги:**

| Флаг | Короткий | Описание | По умолчанию |
|------|----------|----------|--------------|
| `--platform` | `-P` | Платформы публикации (telegram) | telegram |
| `--tag` | `-t` | Теги поста | - |
| `--slug` | `-s` | Slug поста (генерируется автоматически) | из заголовка |
| `--editor` | `-e` | Редактор для открытия файла | `$VISUAL` → `$EDITOR` → `vim` |

**Примеры:**

```bash
# Создать пост с заголовком
jtpost new "Мой первый пост"

# Создать пост с тегами
jtpost new "Обзор Go 1.25" --tag go --tag release

# Создать пост для Telegram
jtpost new "Новости" --platform telegram

# Создать с конкретным slug
jtpost new "Статья" --slug my-article

# Открыть в конкретном редакторе
jtpost new "Заметка" --editor "code --wait"
```

**Формат созданного файла:**
```yaml
---
id: 1772824850782694000-moy-pervyy-post
title: Мой первый пост
slug: moy-pervyy-post
status: idea
platforms:
  - telegram
tags: []
---

Тело поста...
```

### list

Вывод списка постов с фильтрацией.

```bash
jtpost list [флаги]
```

**Флаги:**

| Флаг | Короткий | Описание | По умолчанию |
|------|----------|----------|--------------|
| `--status` | `-s` | Фильтр по статусам | - |
| `--platform` | `-P` | Фильтр по платформам | - |
| `--tag` | `-t` | Фильтр по тегам | - |
| `--search` | `-q` | Поиск по заголовку/slug | - |
| `--format` | `-f` | Формат вывода (table, json) | table |

**Примеры:**

```bash
# Все посты
jtpost list

# Только черновики
jtpost list --status draft

# Посты для блога
jtpost list --platform telegram

# С тегом "golang"
jtpost list --tag golang

# Поиск по заголовку
jtpost list --search "Go"

# Комбинация фильтров
jtpost list --status draft --tag go --platform telegram
```

**Пример вывода:**
```
STATUS  TITLE               SLUG              PLATFORMS  TAGS
------  -----               ----              ---------  ----
idea    Мой первый пост     moy-pervyy-post   telegram   go, cli
draft   Обзор Go 1.25       obzor-go-125      telegram   go

📊 Всего постов: 2
```

### status

Изменение статуса поста.

```bash
jtpost status <id> --set <статус>
```

**Флаги:**

| Флаг | Короткий | Описание |
|------|----------|----------|
| `--set` | `-s` | Новый статус (обязательно) |

**Доступные статусы:**
- `idea` — идея, требует проработки
- `draft` — активная работа над постом
- `ready` — готов к публикации
- `scheduled` — запланирован на дату
- `published` — опубликован

**Правила переходов:**
```
idea → draft → ready → scheduled → published
  └──────→ ready → published
  └───────────→ published
```

Переход возможен только вперёд по жизненному циклу.

**Примеры:**

```bash
# Начать работу над идеей
jtpost status <id> --set draft

# Завершить черновик
jtpost status <id> --set ready

# Опубликовать
jtpost status <id> --set published
```

### publish

Публикация поста на платформу.

```bash
jtpost publish <id> --to <платформа>
```

**Флаги:**

| Флаг | Короткий | Описание | По умолчанию |
|------|----------|----------|--------------|
| `--to` | `-t` | Платформа (telegram) | telegram |
| `--dry-run` | `-d` | Режим предпросмотра без публикации | false |

**Требования:**
- Статус поста должен быть `ready` или `scheduled`

**Примеры:**

```bash
# Опубликовать в Telegram
jtpost publish <id> --to telegram

# Предпросмотр публикации
jtpost publish <id> --to telegram --dry-run
```

**Пример вывода (dry-run):**
```
🔍 Предпросмотр публикации в Telegram
=====================================
📍 Канал: -1009999999999
📝 Заголовок: Мой первый пост
📏 Длина сообщения: 250 символов

📄 Текст сообщения:
-------------------------------------
<b>Мой первый пост</b>

Тело поста в HTML формате...
-------------------------------------

⚠️  Это режим предпросмотра. Для реальной публикации уберите флаг --dry-run
```

### plan

Показать план публикаций на ближайшие дни.

```bash
jtpost plan [флаги]
```

**Флаги:**

| Флаг | Короткий | Описание | По умолчанию |
|------|----------|----------|--------------|
| `--days` | `-d` | Период планирования в днях | 30 |

**Примеры:**

```bash
# План на месяц
jtpost plan

# План на неделю
jtpost plan --days 7

# План на квартал
jtpost plan --days 90
```

**Пример вывода:**
```
DATE        TYPE  STATUS    TITLE              PLATFORMS
----        ----  ------    -----              ---------
2026-02-05  ⏰    ready     Design Patterns    telegram
2026-02-12  📋    scheduled Performance        telegram
2026-02-15  📋    draft     Testing Best...    telegram

📊 Всего постов в плане: 3
```

**Обозначения:**
- `⏰` — запланированная публикация (scheduled_at)
- 📋 — дедлайн (deadline)

### next

Рекомендация следующего поста для работы.

```bash
jtpost next [флаги]
```

**Флаги:**

| Флаг | Короткий | Описание | По умолчанию |
|------|----------|----------|--------------|
| `--format` | `-f` | Формат вывода (full, plain, json) | full |

**Алгоритм рекомендации:**
1. Посты с просроченным дедлайном (наивысший приоритет)
2. Посты с ближайшим дедлайном
3. Посты с ближайшей запланированной публикацией
4. Посты по статусу (ready > draft > idea)

**Примеры:**

```bash
# Полная информация
jtpost next

# Только slug (для скриптов)
jtpost next --format plain

# JSON формат
jtpost next --format json
```

**Пример вывода:**
```
📌 Рекомендуемый пост для работы:
==================================

📝 Заголовок: Обзор Go 1.25
🏷️ Slug: obzor-go-125
📊 Статус: draft
⏰ Дедлайн: 2026-02-10 18:00
🌐 Платформы: telegram
🏷️ Теги: go, release
```

### serve

Запуск HTTP сервера с REST API и Web UI.

```bash
jtpost serve [флаги]
```

**Флаги:**

| Флаг | Короткий | Описание | По умолчанию |
|------|----------|----------|--------------|
| `--addr` | `-a` | Адрес для прослушивания | localhost |
| `--port` | `-p` | Порт для прослушивания | 8080 |

**Примеры:**

```bash
# Запуск на localhost:8080
jtpost serve

# Запуск на всех интерфейсах, порт 3000
jtpost serve --addr 0.0.0.0 --port 3000
```

**Пример вывода:**
```
✅ Telegram publisher инициализирован
🚀 jtpost HTTP сервер запущен на http://localhost:8080
📊 Web UI доступен в браузере
🔌 API endpoints:
   GET    /api/posts      — список постов
   GET    /api/posts/{id} — получить пост
   PATCH  /api/posts/{id} — обновить пост
   DELETE /api/posts/{id} — удалить пост
   POST   /api/posts      — создать пост
   POST   /api/posts/{id}/publish — опубликовать пост
   GET    /api/stats      — статистика
   GET    /api/next       — рекомендация
   GET    /api/plan       — план публикаций

Нажмите Ctrl+C для остановки
```

Web UI построен с использованием [htmx](https://htmx.org) и предоставляет:
- Список постов с фильтрацией
- Статистику по постам
- Рекомендации следующего поста
- Редактирование постов
- Публикацию в Telegram

## Конфигурация

Файл `.jtpost.yaml`:

```yaml
# Директория с постами
posts_dir: content/posts

# Директория с шаблонами
templates_dir: templates

# Настройки Telegram
telegram:
  bot_token: "${TELEGRAM_BOT_TOKEN}"
  chat_id: "${TELEGRAM_CHAT_ID}"

# Настройки по умолчанию
defaults:
  status: idea
  platforms:
    - telegram
```

---

## Переменные окружения

| Переменная | Описание |
|------------|----------|
| `VISUAL` | Предпочитаемый визуальный редактор |
| `EDITOR` | Текстовый редактор по умолчанию |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram бота |
| `TELEGRAM_CHAT_ID` | ID чата/канала Telegram |

## Примеры рабочих процессов

### Создание и публикация поста

```bash
# 1. Инициализация (один раз)
jtpost init

# 2. Создание поста
jtpost new "Как писать тесты в Go" --tag go --tag testing

# 3. Редактирование
jtpost edit <id>

# 4. Пометить как черновик
jtpost status <id> --set draft

# 5. После завершения
jtpost status <id> --set ready

# 6. Публикация
jtpost publish <id> --to telegram
jtpost status <id> --set published
```

### Планирование контента

```bash
# Посмотреть план на месяц
jtpost plan

# Найти посты с дедлайном на этой неделе
jtpost plan --days 7

# Посмотреть все идеи
jtpost list --status idea

# Получить рекомендацию следующего поста
jtpost next
```

### Использование Web UI

```bash
# Запустить сервер
jtpost serve

# Открыть браузер по адресу http://localhost:8080
```

Web UI предоставляет:
- Визуальный список всех постов
- Фильтрацию по статусам и тегам
- Редактирование постов прямо в браузере
- Публикацию в Telegram в один клик
- Статистику и рекомендации
