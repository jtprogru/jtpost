# revision-diff — side-by-side diff с текущей версией

## Goal
На странице ревизии `/ui/posts/{id}/history/{hash}` показывать diff между
выбранной ревизией и текущим состоянием поста, чтобы owner понимал, что именно
изменится при revert.

## User flow
1. Owner открывает revision-page.
2. Вместо одиночного `<pre>` с raw-содержимым видит два колонки:
   - **слева** — содержимое ревизии,
   - **справа** — текущее содержимое.
3. Кнопка «Вернуть к этой версии» остаётся.

## Scope
- ✅ Текстовый side-by-side: line-by-line, без подсветки внутренних различий.
- ✅ Сравниваем только тело файла (markdown) — frontmatter не показываем
  (он содержит UpdatedAt/Revision, которые шумят и всегда отличаются).
- ✅ Diff алгоритм — простой LCS-based, без внешних зависимостей. Pure Go.
- ✅ Подсветка добавленных / удалённых строк через CSS-классы.
- ❌ Word-level diff внутри строк — out of scope.
- ❌ Toggle между unified/split — split всегда.

## Edge cases
- Бинарные файлы / нечитаемое содержимое → отрисовываем как текст (FileAtCommit
  для постов всегда возвращает .md, риск минимальный).
- Текущий пост ещё не имеет версии в истории (новый файл) → diff = весь контент
  справа добавлен.

## Non-functional
- 0 lint, race-clean.
- Без `goldmark` и без render markdown — только plaintext построчный сравнитель.
